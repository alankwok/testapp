uHub 超方便, 雲端免費自動轉換影片格式

上星期介紹完申請免費uHub 戶口， 說過會來一篇實測， 今天將寫了一半的文章完成。

於實測之前又再來說說故事...... 話說於新浪微博轉貼上一篇有關uHub 開戶口文章的時侯，  有兩三個童鞋倒轉來介紹我用 goaruna 及 minus 兩家外地的雲端儲存服務， 當談到uHub 不單純是雲儲存服務， 而且有影片格式"自動轉換功能"的時候， 兩位童鞋便自動投降默默無聲了。
::

::
"自動轉換功能" 是PCCW uHub 自家提供的功能， 用家將不同格式的影片利用它便可以將不同影片格式的檔案， 變成任何便攜工具如 iPhone、iPad 、Android等手機或平板電腦能夠播放的格式，所以說到 uHub 並不單是雲儲存服務是有它的原因的。

另外如要再比較其他雲儲存服務， uHub 都再多有一個優點， 就是他們的雲儲存庫都是建基於本地的數據中心，大家都知道香港的 ISP 互聯網服務供應商對於本地的數據速度， 與及瀏覽外地的速度(所謂出海的速度) 都有著很大的差距， 瀏覽本地的速度往往快過瀏覽外地數倍之多， 所以uHub設於本地的數據中心， 就有著先天性的速度優勢。



::

往下去我再測試一下 uHub 的 iPhone 軟件的便利性， 並作一點簡單的評測。uHub iPhone軟件設計與 Android 上的基本是一樣， 除了開始的主頁面(↓左圖)， 便會自動轉到主功能頁(↓右圖)， 當中簡潔地指示用家四種不同的分類形式， 如音樂、電影、相片及全部。下方則主要是中間那兩個功能 - "搜尋" 及 "上載"。


::

上載的方式的都是超簡單的， 如下圖一樣， 找個類別及方式如 "Photo/Video from library" ， 再跳到 iPhone 的 Camera Roll 選擇相片， 就可以上載到uHub， 上載速度是超快的， 可能是因為每次一張及不用壓縮的原故， 而且用 uHub 有一好處就是不會像 Dropbox 那樣將圖片自行壓縮， 所以不會無源無故地弄醜了圖片。若然能夠同時上載多張圖片， 相信工序會更為流暢， 用戶體驗會更好。

::

再試試播放 uHub 上的影片， 基本測試了原裝的uHubUserGuide.wmv，   *.WMV是 Microsoft windows 的專用影片格式， 亦即是代表不能在iPhone 上直接播放， 所以用它來測試就正好， 可以看到uHub 從轉換影片格式到播放畫面所需時間。從按鍵那刻開始， 當中經過轉檔及解壓等步驟， 直至看到影片的圖像時， 10.53mb 檔案都只是用了三五秒便看到畫面的圖像， 我覺得算是接近實時播放， 用家實際使用時， 純對不會有久等的壞感覺。



::

↓ 我又另外用了 motorola milestone 3 安裝Android 的 uHub 軟件來測試看雲端上影片的效果， 今次手機服務供應商則用改為 One2Free。測試的檔案用了 What is uHub.mov，   要播放 1分34秒的 *.mov 檔用了約十秒時間， 雖然不明白為什麼時間長了一點， 但整體都感覺快速。



::

↓ 而我亦有用 iPhone 直接拍攝的影片在手， 所以試了上載到uHub 當中， 此手機服務供應商是smartone， 再配合 PCCW 的 uHub， 上載速度都算是速度很快的組合， 2分鐘影片壓縮時間用了1分30秒，  上載時間實際用了4分鐘左右， 我覺得算是合理的表現。唯獨是有些影片上載到uHub 後會有下圖的效果出現， 大家要留意一下。



::

↓ 而查看 uhub 上的文字檔是超快的， 可能是因為我用了純文字檔來測， 檔案大小只有數 kb 大小， 運作十分流暢。


影片中大家又可以發現 uHub 的Andorid 軟件打開文字檔時會問那一個閱讀器， 當中亦有Quick Office 可選擇， 因此看出用什麼閱讀器來打開檔案， 其賓係基於手機有什麼軟件可選， 這情況應與uHub 無關。



::

最後提到 uHub 另外一個問題， 就是沒有網頁版本的工作介面。 好似我的情況， 公司中有一此舊電腦仍然運行 Windows 2000， 所以不能夠安裝uHub的免費電腦程 式，將檔案上載至uHub雲端伺服器， 此時若有網頁版本的話就可以解決， 雖然只是小問題， 但始終我認為這類Web application 服務的網頁版本是頗重要， 希望PPCW 能夠將這少少的缺失補完。
